Arquitectura de Cloud Computing

Actividad Openstack para BONARD

Fernando Quevedo - Octubre 2024

Esta presentación detalla la arquitectura de cloud computing diseñada para BONARD, una empresa con múltiples sucursales y necesidades específicas de infraestructura IT.

Componentes y Servicios

BONARD requiere una infraestructura compleja que incluye varios componentes clave. En la oficina central, necesitamos:

  • Servidor de respaldo de archivos: Para garantizar la seguridad de los datos.
  • Servidor de facturación general: Centraliza todas las operaciones financieras.
  • Servidor de cámaras: Para gestionar la seguridad visual de todas las locaciones.
  • Servidor de Active Directory: Gestiona usuarios y políticas de acceso.
  • Servidor web: Aloja la página web estática de la empresa.
  • Servidor FTP: Facilita la transferencia segura de archivos.
  • Servidor de base de datos: Almacena y gestiona todos los datos críticos.
  • Servidor de correo electrónico: Maneja las comunicaciones internas y externas.

Diseño en Google Cloud

Para implementar esta infraestructura en Google Cloud, aprovecharemos varios servicios clave:

  • Google Cloud Storage: Utilizaremos este servicio para el respaldo de archivos y almacenamiento de videos de las cámaras. Ofrece alta durabilidad y acceso rápido a los datos.
  • Compute Engine: Para los servidores de facturación, web, y FTP. Estas instancias virtuales nos dan la flexibilidad para configurar los recursos según nuestras necesidades específicas.
  • Cloud SQL: Ideal para nuestras bases de datos, ofreciendo alta disponibilidad y fácil escalabilidad.
  • VPC (Virtual Private Cloud): Nos permite crear una red privada y segura para todos nuestros servicios, con control total sobre la topología de red.
  • IAM (Identity and Access Management): Crucial para gestionar los accesos y permisos de manera granular en toda nuestra infraestructura cloud.

Implementación por Localización

La arquitectura se divide entre la oficina central y las sucursales para optimizar el rendimiento y la gestión:

Central

La oficina central albergará los servicios principales, incluyendo el respaldo centralizado, la facturación general, y los servidores críticos como Active Directory y correo electrónico. Esto permite una gestión centralizada y una mayor seguridad para los sistemas críticos.

Sucursales

Cada sucursal tendrá servidores locales para facturación, cámaras y bases de datos. Esto mejora la velocidad de acceso a los datos locales y proporciona cierta autonomía. Algunas sucursales también tendrán servidores de relojes para gestión de tiempo específica.

Fundamentos de las Decisiones

Nuestras decisiones de diseño se basan en varios principios clave:

  • Escalabilidad: La arquitectura en Google Cloud permite aumentar o disminuir recursos según las necesidades, asegurando que BONARD pueda crecer sin problemas.
  • Seguridad integrada: Utilizamos las herramientas de seguridad de Google Cloud como IAM y VPC para proteger los datos y el acceso a los sistemas.
  • Resiliencia y alta disponibilidad: Los servicios de Google Cloud ofrecen redundancia y replicación para minimizar el tiempo de inactividad.
  • Economía de costos: El modelo de pago por uso permite optimizar los costos, pagando solo por los recursos que realmente se utilizan.

Tipos de Flavours en Cloud Computing

En cloud computing, los "flavours" o tipos de instancias son configuraciones predefinidas de recursos. Para BONARD, consideraremos los siguientes tipos:

General Purpose

Equilibrio entre CPU y RAM, ideal para servidores web y aplicaciones ligeras. Usaremos estos para los servidores FTP y web estática de BONARD.

Compute Optimized

Alta CPU y baja RAM, perfecto para procesamiento intensivo. Recomendado para los servidores de facturación de BONARD, que requieren cálculos frecuentes.

Memory Optimized

Alta RAM y baja CPU, óptimo para bases de datos y análisis en memoria. Utilizaremos estos para los servidores de bases de datos centrales y de sucursales de BONARD.

Storage Optimized

Alto almacenamiento y E/S, ideal para el servidor de cámaras de BONARD que maneja grandes volúmenes de datos de video.

GPU Optimized

Incluye GPUs para procesamiento gráfico. No es necesario para las necesidades actuales de BONARD, pero podría ser útil en el futuro para análisis de video avanzado.

High I/O

Alto rendimiento de entrada/salida, potencialmente útil para el servidor de base de datos central de BONARD si se requiere un alto volumen de transacciones.

Configuración de Red

La configuración de red es crucial para la seguridad y eficiencia de la infraestructura de BONARD. Implementaremos lo siguiente:

  • VPC única: Crearemos una Virtual Private Cloud (VPC) que abarcará toda la infraestructura de BONARD. Esto nos permite tener un control centralizado sobre la red y sus políticas.
  • Subredes por localización: Dentro de la VPC, configuraremos subredes específicas para la central y cada sucursal. Esto mejora la organización y permite aplicar políticas de seguridad más granulares.
  • Reglas de firewall específicas: Implementaremos reglas de firewall para controlar el tráfico entre subredes y hacia/desde Internet. Por ejemplo, limitaremos el acceso a los servidores de base de datos solo desde los servidores de aplicaciones autorizados.
  • Routing entre subredes: Configuraremos rutas para permitir la comunicación necesaria entre la central y las sucursales, asegurando que los datos fluyan de manera eficiente y segura.
  • Aislamiento de componentes sensibles: Utilizaremos grupos de seguridad y ACLs de red para aislar componentes críticos como las bases de datos y los servidores de Active Directory, añadiendo una capa adicional de seguridad.

Seguridad y Monitorización

La seguridad y la monitorización son aspectos críticos para proteger los activos digitales de BONARD y garantizar el rendimiento óptimo de la infraestructura:

  • IAM para control de acceso: Utilizaremos Identity and Access Management para gestionar de forma granular quién tiene acceso a qué recursos. Esto es especialmente importante para los servidores críticos como los de facturación y bases de datos.
  • Reglas de firewall ingress/egress: Implementaremos reglas estrictas de entrada y salida para controlar el tráfico de red. Por ejemplo, limitaremos el acceso SSH solo a IPs autorizadas y restringiremos el tráfico saliente de los servidores de base de datos.
  • Logging de firewall activado: Activaremos el logging detallado de las actividades del firewall. Esto nos permitirá auditar el tráfico de red y detectar posibles amenazas o comportamientos anómalos.
  • VPC Flow Logs: Habilitaremos VPC Flow Logs para obtener información detallada sobre el tráfico de red. Esto es crucial para el análisis de seguridad, resolución de problemas y optimización del rendimiento de la red.
  • Monitorización continua: Implementaremos herramientas de monitorización en tiempo real para supervisar el rendimiento de los servidores, el uso de recursos y la salud general de la infraestructura. Esto nos permitirá detectar y responder rápidamente a cualquier problema.

Conclusiones y Recomendaciones

Basándonos en el análisis y diseño realizado para BONARD, podemos concluir lo siguiente:

  • Arquitectura escalable y segura: La solución propuesta en Google Cloud ofrece a BONARD una infraestructura que puede crecer con la empresa, manteniendo altos estándares de seguridad.
  • Aprovechamiento de servicios cloud: Al utilizar servicios gestionados como Cloud SQL y Cloud Storage, BONARD puede reducir la carga operativa y centrarse en su negocio principal.
  • Alta disponibilidad y redundancia: La arquitectura diseñada asegura que los servicios críticos de BONARD estén siempre disponibles, con redundancia incorporada en los componentes clave.
  • Optimización de costos: El modelo de pago por uso de Google Cloud permite a BONARD optimizar sus gastos en TI, pagando solo por los recursos que realmente utiliza.
  • Gestión centralizada: Con herramientas como IAM y la VPC unificada, BONARD puede gestionar de manera eficiente su infraestructura distribuida desde un punto central.

Recomendamos a BONARD implementar esta arquitectura en fases, comenzando con los servicios críticos y expandiendo gradualmente. Además, es crucial mantener un programa de formación continua para el personal de TI para aprovechar al máximo las capacidades de Google Cloud.
